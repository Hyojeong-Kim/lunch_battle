# Lunch War 태그(Tag) 스펙 v1

문장 풀(Claim/Reason/Style)의 태그를 **일관된 규칙**으로 관리하기 위한 공식 스펙 문서.  
UI/랜덤 선택/점수 계산/컨텍스트 매칭에 모두 사용된다.

---

## 1. 태그 목적

태그는 다음 용도로 사용된다:

1. **컨텍스트(weather/weekday/salary)와 문장 궁합 계산**
2. 랜덤 뽑기 시 문장 다양성 확보
3. 문장 톤/상황/뉘앙스 정리(메타데이터 역할)
4. 카테고리 확장 시 유지보수 편의성

---

## 2. 태그 표기 규칙

> **`카테고리_값` 형태로 작성한다.**  
> 예: `날씨_비`, `상태_지침`, `상황_야근`

- 띄어쓰기 없음
- 한글 사용
- 값은 가능한 한 **짧고 의미 명확**하게
- 컨텍스트 매칭과 무관한 감정/상황 태그도 동일 규칙 적용 (단, v1에서는 비활성)

---

## 3. 태그 카테고리 목록 (v1에서 실제 사용되는 것만)

| 구분             | 태그 prefix | 예시 값          | 설명                                |
|------------------|------------|------------------|-------------------------------------|
| 날씨(Weather)    | `날씨_*`   | 맑음/흐림/비/눈  | 게임 시작 시 컨텍스트와 직접 매칭  |
| 요일(Weekday)    | `요일_*`   | 월/화목/수/금    | 사무실 분위기/일과 리듬            |
| 월급 타이밍(Salary) | `월급_*` | D-5/월급날/D+5   | 지갑 상황과 소비 성향              |

### 3-1. 날씨(Weather)

- `날씨_맑음`
- `날씨_흐림`
- `날씨_비`
- `날씨_눈`

### 3-2. 요일(Weekday)

- `요일_월`
- `요일_화목`
- `요일_수`
- `요일_금`

### 3-3. 월급 타이밍(Salary Timing)

- `월급_D-5`
- `월급_월급날`
- `월급_D+5`

---

## 4. 상황(Situation) 태그

**점수에 직접 영향이 가지 않는 모든 상황 태그는 제거한다.**

> 현재 버전(v1)에서는 상황 태그를 **사용하지 않는다.**

---

## 5. 상태(Condition) 태그

**점수에 직접 영향이 가지 않는 모든 상태 태그는 제거한다.**

> 현재 버전(v1)에서는 상태 태그를 **사용하지 않는다.**

---

## 6. 가치(Value) 태그

**점수에 직접 영향이 가지 않는 모든 가치 태그는 제거한다.**

> 현재 버전(v1)에서는 가치 태그를 **사용하지 않는다.**

---

## 7. 태그 사용 규칙 (최소 버전)

1. **문장에 태그는 최대 1~2개만 사용하며, 오직 아래 3종만 사용한다.**
   - `날씨_*`
   - `요일_*`
   - `월급_*`
2. 모든 다른 상황/상태/가치 태그는 v1에서는 사용하지 않는다.
3. 점수 계산은 현재 컨텍스트(weather/weekday/salary)와 태그의 일치 여부만으로 판단한다.

---

## 8. 문장에 적용되는 태그 예시 (축약형)

```json
{
  "id": "claim_korean_02",
  "tone": "감성형",
  "tags": ["요일_금"]
}
```

```json
{
  "id": "reason_korean_03",
  "tone": "논리형",
  "tags": ["월급_D-5"]
}
```

```json
{
  "id": "style_korean_01",
  "tone": "논리형",
  "tags": ["날씨_비"]
}
```

---

## 9. 목적 요약

현재 v1에서는 시스템을 단순화하기 위해  
**점수에 직접 영향이 가는 태그(날씨/요일/월급)**만 사용한다.

모든 상황/상태/가치 태그는 제거하여 유지보수를 단순화하고,  
문장 길이/구조 모두 경량화한다.
